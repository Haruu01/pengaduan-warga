GitHub Integration

Simpan pengaduan sebagai GitHub Issues untuk transparansi dan akses publik

Coba Sekarang Panduan Setup

Keunggulan GitHub Integration

Mengapa menggunakan GitHub Issues untuk menyimpan pengaduan?

Transparansi Publik

Semua pengaduan dapat dilihat publik di GitHub repository, meningkatkan transparansi pemerintahan.

Riwayat Lengkap

GitHub menyimpan semua perubahan status dan komentar dengan timestamp yang akurat.

Diskusi Terbuka

Masyarakat dapat memberikan komentar dan diskusi terkait pengaduan yang diajukan.

Kategorisasi

Penggunaan labels untuk mengkategorikan pengaduan berdasarkan jenis dan status.

Pencarian Mudah

Fitur pencarian GitHub yang powerful untuk menemukan pengaduan spesifik.

Notifikasi Real-time

Notifikasi otomatis via email ketika ada update pada pengaduan.

Panduan Setup

Ikuti langkah berikut untuk mengaktifkan GitHub Integration

1
Buka GitHub Settings

Masuk ke akun GitHub Anda dan buka halaman Personal Access Tokens

Buka GitHub Settings
2
Generate New Token

Klik "Generate new token (classic)" dan isi form berikut:

  • Note: "Pengaduan Warga App"
  • Expiration: Pilih sesuai kebutuhan
  • Scopes: Centang "public_repo"
Penting: Pastikan scope "public_repo" dipilih agar aplikasi dapat membuat issues.
3
Copy Token

Setelah token dibuat, copy token tersebut. Token akan terlihat seperti ini:

ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Keamanan: Jangan bagikan token ini kepada siapapun!
4
Setup di Aplikasi

Buka aplikasi dan masukkan token di pengaturan GitHub Integration

  1. Login ke aplikasi
  2. Klik tombol "Setup GitHub" di dashboard
  3. Paste token yang sudah dicopy
  4. Klik "Simpan Token"
Buka Aplikasi

Cara Kerja

Bagaimana pengaduan disimpan sebagai GitHub Issues

Pengaduan Baru

Ketika warga membuat pengaduan baru:

  • Data pengaduan diformat sebagai GitHub Issue
  • Judul: "[PENGADUAN] Judul Pengaduan"
  • Labels: "pengaduan", "kategori:xxx", "status:pending"
  • Body berisi detail lengkap pengaduan
Update Status

Ketika admin mengubah status:

  • Label status diperbarui (pending/process/completed)
  • Komentar baru ditambahkan dengan tanggapan admin
  • Timestamp otomatis tercatat
  • Notifikasi dikirim ke pelapor

Contoh GitHub Issue

Begini tampilan pengaduan di GitHub

[PENGADUAN] Jalan Rusak di Depan Sekolah #1
pengaduan kategori:infrastruktur status:pending
Detail Pengaduan
Pelapor: Ahmad Rizki
Email: ahmad@email.com
Kategori: Infrastruktur
Lokasi: Jl. Pendidikan No. 123

Jalan di depan SD Negeri 1 mengalami kerusakan parah dengan banyak lubang yang membahayakan pengendara.
